# Redis Key 네이밍 컨벤션

Redis에서 Key 이름을 적절하게 설계하는 것은 데이터 관리의 효율성을 결정짓는 매우 중요한 요소다. 현업에서 널리 사용되는 주요 네이밍 컨벤션을 정리한다.

### ✅ 현업에서 자주 활용하는 네이밍 컨벤션

다양한 컨벤션이 존재할 수 있으나, 가장 보편적으로 사용되는 규칙은 다음과 같다.

> **콜론(`:`)을 활용해 계층적으로 의미를 구분하여 사용**

#### 💡 예시를 통한 이해
- `users:100:profile`: 사용자들(users) 중에서 PK가 100인 사용자(user)의 프로필(profile)
- `products:123:details`: 상품들(products) 중에서 PK가 123인 상품(product)의 세부사항(details)

이러한 규칙은 고정된 것이 아니며, 서비스의 요구사항이나 팀의 규칙에 따라 응용하여 사용할 수 있다.

### ✅ 계층적 네이밍 컨벤션의 장점

이러한 방식으로 Key를 정의하면 다음과 같은 여러 이점을 얻을 수 있다.

1. **가독성**: 데이터의 의미와 용도를 한눈에 파악할 수 있다.
2. **일관성**: 일정한 규칙을 따름으로써 코드의 일관성이 높아지고 유지보수가 용이해진다.
3. **검색 및 필터링 용이성**: 패턴 매칭을 사용하여 특정 유형의 Key를 쉽게 검색하거나 대량으로 관리할 수 있다.
4. **확장성**: 서로 다른 데이터 간에 Key 이름이 겹쳐 충돌할 위험이 현저히 낮아진다.
